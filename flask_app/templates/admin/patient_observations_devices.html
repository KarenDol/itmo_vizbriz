📊 Level 4 Reports: Observations & Device Recommendations

Clinical data analysis and device design patterns

What is this page showing?

This table shows all processed Level 4 sleep study reports and the device recommendations that were made for each patient.

  • Each row = One report (or one device context from a report)
  • Report column: The filename of the Level 4 report that was uploaded
  • Patient Name & ID: Patient information (if the report was matched to a patient in the system)
  • Dentist: The dentist who treated this patient (if patient was matched)
  • Context: The type of device context (e.g., "Nighttime MAD" or "Daytime TMJ")
  • Blue columns (Observations): What was found in the sleep study (AHI, O2 levels, obstruction sites, etc.)
  • Green columns (Device Design): What device parameters were recommended based on the observations
  • Orange column (Device Options): Which specific devices were recommended

Purpose: This helps you see patterns - how different clinical observations lead to different device recommendations. You can use this to build decision trees for future cases.

📋 How Context is Determined:

The Context column shows the device type context extracted from the report:

  • "Nighttime MAD" - Found when report mentions "Nighttime", "MAD", or "Mandibular Advancement Device"
  • "Daytime TMJ" - Found when report mentions "Daytime", "TMJ", or "Lower TMJ Appliance"
  • "Unknown" - Used when the context is unclear or not explicitly stated

The LLM extraction service analyzes report sections and looks for these keywords to determine the context. One report can have multiple contexts (e.g., both nighttime and daytime devices), which creates multiple rows.

ℹ️ About Empty Fields:

Empty fields (shown as "Not found") can mean:

  • The information wasn't in the original report - Some reports may not include all sections (e.g., patient complaints, clinical background)
  • The extraction parser couldn't find it - The LLM may have missed the information if it was formatted differently than expected
  • Check the "Extraction Status" column - Shows HIGH/MED/LOW confidence based on how many fields were successfully extracted

Tip: Hover over empty cells to see if extraction was attempted. If you see many empty fields with LOW confidence, the report may need reprocessing.

📥 How to Add More Reports:
  • Option 1 (Recommended - Web UI): Click "Process Reports from Admin Files" button above. This will automatically find all Level 4 reports in the admin_files table, download them from S3, and process them.
  • Option 2 (Command Line - Directory): Run: python scripts/process_l4_reports.py --input "/path/to/reports/"
  • Note: Reports can be in PDF or DOCX format. The system will extract text, parse observations (AHI, O2 levels, obstruction sites, etc.), device design parameters, and device options. All observations are stored in the l4_device_design table. Already processed reports will be skipped.
